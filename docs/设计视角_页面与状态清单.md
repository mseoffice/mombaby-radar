## 2. 状态清单（状态机）

### 2.1 content 状态机（34.2.6，已确认）

```
        ┌─────────────┐
        │  generating  │  Agent 生成完成
        └──────┬──────┘
               │
               ▼
        ┌──────────────┐   人工通过   ┌──────────┐
        │ pending_review│──────────▶│ approved  │
        └──────┬──────┘             └────┬─────┘
               │ 人工驳回                   │ 标记已发布
               ▼                           ▼
        ┌──────────┐                ┌──────────┐
        │ rejected │──重新生成─────▶│ published │──下线/归档──▶ archived
        └──────────┘                └──────────┘
```

| 当前状态 | 触发事件 | 目标状态 |
|---|---|---|
| generating | Agent 生成完成 | pending_review |
| pending_review | 人工通过 | approved |
| pending_review | 人工驳回 | rejected |
| rejected | 重新生成/重新提交 | generating → pending_review |
| approved | 标记已发布 | published |
| published / archived | 下线或归档 | archived |

### 2.2 建议补充定义的状态（草案，待设计评审确认）

| 实体 | 建议状态 | 说明 |
|---|---|---|
| agent_task | pending → running → success / failed / partial | Agent 任务执行态；partial 用于超时被标记 |
| comment | new → analyzed → replied / pending | 评论处理态 |
| 群推送任务 | draft → queued → sent → recorded | 推送计划执行态 |
| 商品导入任务 | uploading → imported / failed | Webhook/批量导入态 |

> 以上为建议草案，最终状态需在"数据模型 ER 终稿 + 状态清单终稿"中由设计+后端确认。

## 3. 交互边界（硬约束）

| 边界 | 约束 | 来源 |
|---|---|---|
| AI 内容标识 | AI 生成内容**必须标注"AI 生成"标识**（全局可见） | 合规（附录C）/ 附录B EARS |
| 发布锁定 | 内容状态为 `pending_review` 时，**锁定"发布"按钮** | State-driven（附录B） |
| 人工审核 | **人工审核环节必须保留**，不可全自动 | 假设 A4 |
| 失败降级 | LLM/图片/平台 API 失败 → 明确兜底提示 + 通知运营 | 附录B Unwanted 示例 |
| 一键复制 | 发布采用"一键复制 + 手动粘贴"（25.2），需提供复制成功反馈 | 25.2 发布流程 |
| 采集方式 | 评论/私信采集方式待定（D3），界面预留手动粘贴入口 | D3 / 26.1 |

**空态 / 加载态 / 异常态规范（建议）**
- 空态：无商品 / 无内容 / 无评论 / 无数据时给出引导操作（如"导入第一个商品"）。
- 加载态：内容生成中、评论分析中显示骨架屏 + 进度；禁止空白卡死。
- 异常态：生成失败提供"重试"；平台 API 限流提示"稍后重试"；网络异常全局 toast。

## 4. 设计验收 Checklist

### 4.1 全局
- [ ] 所有 AI 生成内容带"AI 生成"标识
- [ ] 首屏加载 ≤2s、页面切换 ≤1s（第 34 章）
- [ ] 空态 / 加载态 / 异常态三态齐全
- [ ] 个人信息展示/留存符合 35.2（脱敏、最小化）
- [ ] 响应式与浏览器兼容（第 34.5）

### 4.2 分页面（来自附录B 验收标准）

| 页面 | 关键验收点 |
|---|---|
| P1 Dashboard | 6 卡片数据 T+1 更新；待办含评论/好价；趋势图正确；AI 建议可折叠 |
| P2 商品 | 导入（Webhook/手动/批量）可用；价格历史记录；CRUD 正常 |
| P3 内容工厂 | ≤3 分钟生成 5 平台；规格符合 12.3；状态机正确；pending_review 锁定发布 |
| P4 AI 运营 | 评论分析 ≤5s；生成 2–3 回复方案；私信商品匹配 |
| P5 私域 | 推送计划生成；一键复制发送；效果记录 |
| P6 数据 | 日报/周报生成；商品/内容/评论三维分析正确 |

### 4.3 非功能（第 35 章）
- [ ] 性能：首屏 ≤2s、API ≤200ms、内容生成 ≤3min、图片 ≤30s、评论分析 ≤5s、50 并发
- [ ] 安全：JWT 2h 可刷新；操作审计保留 6 个月（operation_log）
- [ ] 可用性：关键路径降级可用（见研发大纲第 6 章）
