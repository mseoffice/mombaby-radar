## 4. 关键依赖

### 4.1 外部依赖（38.2）

| 依赖项 | 提供方 | 接入方式 | 风险 |
|---|---|---|---|
| 第三方比价机器人 | 外部服务 | Webhook | 服务中断、数据格式变更（A1） |
| LLM API | OpenAI / 百度 / 阿里 | RESTful | 限流、不可用、成本波动（A2） |
| 图片生成 API | OpenAI / Stability AI | RESTful | 质量不稳、生成失败（A6） |
| 平台开放 API | 小红书 / 抖音 / 微博 / 微信 | API（多平台） | 鉴权、限流、规范变更（D2 多平台接入） |

### 4.2 关键假设（38.1，摘要）

- **A5**：V1.0 用户规模 ≤ 50 人，单体架构可满足性能（超出则需微服务拆分）。
- **A7（已确认）**：单团队内部工具，无多租户隔离，数据模型不引入 `tenant_id`。
- A2/A3：LLM API 持续可用、平台规范周期内不变——设计需预留降级开关。
- A4：用户接受"AI 生成 + 人工审核"模式——**人工审核环节必须保留**，不可全自动。

---

## 5. 风险点（38.3）

- **中高风险项**：LLM 质量不稳定、平台 API 波动、比价机器人中断、API 成本不可控。
- **排期风险**：已识别多项中高风险，但排期**未留 buffer**（38.3 原文提示）。
- **应对建议**：
  - 在关键里程碑（第 4 周、第 7 周）间预留 **1 周缓冲**；
  - 或 V1.0 范围做减法（图片生成、群数据看板降为非核心）；
  - 每个外部依赖准备**降级方案**（如 LLM 连续失败 3 次降级人工兜底并通知运营，见附录B EARS 示例）。

| 数据看板页 | 33.1 | 报表/图表 |
| （Dashboard 页面布局规范） | 10.4 | 布局基线 |

### 6.2 状态清单（需设计落地的状态）

- **content（主）**：`generating → pending_review → approved → published → archived`；`rejected → generating`。
- **建议补充定义**（设计中明确）：`agent_task` 状态、`comment` 状态、群推送任务状态、商品导入任务状态。

### 6.3 交互边界（硬约束）

- AI 生成内容**必须标注"AI 生成"标识**（合规，附录C / 附录B）。
- 内容状态为 `pending_review` 时，**锁定"发布"按钮**（State-driven）。
- **人工审核环节必须保留**，不能完全自动化（A4）。
- 异常/失败需有明确兜底提示与通知（如 LLM 失败降级人工）。

### 6.4 设计验收点（来源：附录B + 第 35 章）

- Dashboard：看板 T+1 更新、待办含评论/好价、趋势图正确（P0）。
- 内容工厂：≤3 分钟生成 5 平台、规格符合 12.3、状态机正确（P0）。
- AI 运营中心：评论分析 ≤5s、生成 2–3 回复方案、私信商品匹配（P0）。
- 非功能：性能/安全/可用性达标（第 35 章，P0）。

## 7. 研发视角

### 7.1 技术依赖与选型（第 7 章）

- 前端 Vue3；后端 Spring Boot；AI Agent + RAG 知识库；图片生成接入外部 API。
- 部署：单体架构（A5，≤50 人），INTELLIGENT_TIERING 存储。

### 7.2 接口问题

- 多平台 API **鉴权/限流/规范差异**（D2 多平台接入，Q5 待技术闭环）。
- Webhook 接收好价数据（A1），需幂等处理与格式校验。
- 图片/LLM API 失败重试与降级（附录B Unwanted 示例）。

### 7.3 数据结构

- 核心表：`product` / `content` / `agent_task` + 补充表（role、content_review_log、operation_log，P1-5）。
- content 状态机须落到 `content.status` 枚举与流转逻辑（34.2.6）。
- **A7 约束**：不引入 `tenant_id`，按单团队简化。

### 7.4 兼容性与实现风险

- 平台规范变更（A3）→ 内容模板需可热更新。
- LLM 不稳定（A2）/ 成本（A6）→ 限流、缓存、成本监控。
- 性能基线（第 34 章）：首屏 ≤2s、API ≤200ms、内容生成 ≤3min、图片 ≤30s/全套 ≤2min、评论分析 ≤5s、50 并发、容量 10 万商品+50 万内容+100 万评论。
